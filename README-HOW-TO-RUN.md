# Gower Websites + Applications backend

Two brand websites (Gower Living and Gower Capital Group) served by one small Node app,
sharing a private **applications** backend with automatic screening and a staff dashboard.

```
public/living      gowerliving.com         — home + /apply
public/capital     gowercapitalgroup.com   — home + /apply
public/admin.html  /admin                  — private staff dashboard
server.js          the web server + API
screening.js       the screening rules (your criteria, editable)
data/              applications are saved here as JSON
```

## Run it on your Mac

1. **Install Node.js** (one-off): download the **LTS** from https://nodejs.org and install.
2. **Easiest:** double-click **`start.command`** in this folder. It installs what's needed,
   starts the server and opens the site. (First run may take a minute.)
   **Or** from Terminal:
   ```
   cd "Gower-Websites"
   npm install
   npm start
   ```
3. Open in your browser:
   - Gower Living  → http://localhost:3000/living
   - Capital Group → http://localhost:3000/capital
   - Apply form    → http://localhost:3000/living/apply  (or /capital/apply)
   - Admin         → http://localhost:3000/admin  (passcode: **gower** — change this, see below)

Submitted applications appear instantly in the Admin dashboard, auto-screened.

## The admin dashboard
- **Tabs:** Active (New + Reviewing), Offers, **Rejected**, Archived, All. Rejected applicants
  move into their own list.
- **Status** per applicant (New → Reviewing → Offer made → Rejected → Archived), changed in the
  detail panel.
- **Filters:** recommendation (Shortlist / Serious flag / Borderline / Exclude), site, name search.
- **Export CSV** of everything.
- **Recommendations** are generated by your rules in `screening.js`: exclude many convictions or
  active drug dependency; prefer PIP; under-35 must have 3+ months in temporary accommodation.
  Edit that file to tune the rules.

## Going live on gowerliving.com and gowercapitalgroup.com
The app picks the brand automatically from the domain name, so **one deployment serves both sites**.

Simplest route (managed host, gives HTTPS automatically):
1. Put this folder in a Git repo (GitHub).
2. Create a free/low-cost web service on **Render.com** or **Railway.app**, connect the repo.
   Build command `npm install`, start command `npm start`.
3. Add a **persistent disk** mounted at `/data` (so applications aren't lost on redeploy), or move
   to a database (see below).
4. Point both domains at the service (CNAME), and add both as custom domains so HTTPS covers them.

Set these environment variables in production:
- `ADMIN_PASSWORD` — your staff passcode (do **not** keep the default).
- `SESSION_SECRET` — a long random string.
- `NODE_ENV=production` — makes the login cookie secure (HTTPS-only).

## Important — this holds sensitive personal data
Applications include health, drug, and criminal-record information (special-category and
criminal-offence data under UK GDPR). Before real applicants use it:
- Serve over **HTTPS only** and change the admin passcode.
- Restrict who can reach `/admin`; consider per-staff logins.
- Add a **privacy notice**, a **retention policy**, and run a short **DPIA**.
- Back up the `data/` folder (or use a managed database).
- For more than light use, move storage from the JSON file to a database (e.g. SQLite/Postgres).
